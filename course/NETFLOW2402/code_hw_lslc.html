3rd coding assigment
Due 05/18 (Tue) 14:00

You are asked to write 3 programs which can

  • read the following network file: test1.sp
  • 1st program (spheap1): use Dijkstra's binary heap implementation
  • 2nd program (spdial1): use Dijkstra's Dial's implementation
  • 3rd program (spbf1): use Bellman-Ford's FIFO implementation
  • all 3 programs allow the user to input a source node index s, then output a shortest path with its length from s to all the other nodes.(i.e. a 1-ALL shortest path problem)

Screen Input: network filename (e.g. test1.sp), s (source node)
Screen Output: for each other node, printout a shortest path with its length from s

example: suppose your code is named as spheap1(& spdial1), then when you execute spheap1(& spdial1), you should see:

Please input network filename: (then you input, say, test1.sp)
Please input a source node: (then you input, say, 2)

then you should see the output like

2->1: [can not reach]
2->3: [2] 3<-2
2->4: [2] 4<-2
2->5: [4] 5<-3<-2
2->6: [7] 6<-5<-3<-2

Likewise, if you input source node as node 3, it should output

3->1: [can not reach]
3->2: [can not reach]
3->4: [1] 4<-3
3->5: [2] 5<-3
3->6: [5] 6<-5<-3

What's difficult in this assignment may be

  1. how to do binary heap operations (textbook page 115):
    Basically we need a min-heap data structure, and we need to "create-heap", "find-min", and "delete-min" heap operations. So you may had better read some data structure or algorithm books about how to implement a binary min-heap and its operations.
  2. how to do Dial's bucket (page 113 & MIT animation slides):
    you should first understand how Dial's implementation works. I suggest you to read MIT's slides first. Since Dial's bucket corresponds to integer lengths, you will have to change your code so that each arc length is an integer instead of real number. Dial's implementation should be easier than the binary min-heap implementation.
  3. how to do Bellman-Ford's queue implementation:
    this should be no problem for you since you have done BFS before. Take a closer look at page 142, then you should be able to do it.

2nd coding assignment
Due 03/20 (Mon) 14:00
 

You are asked to write a program that can

  • read a network file (with a format similar to test1.sp )
  • ask the user to input a source node (s) and a sink node (t)
  • print out all the reachable nodes from s by the BFS labeling order, and the distance label associated with each reachable node from s
  • print out all the paths and their lengths from s to t, based on the idea of DFS; mark a shortest path with "*".

Screen Input: network filename (e.g. test1.sp), s (source node), t (sink node)
Screen Output: (1) all the reachable nodes and their distance labels from s by the BFS labeling order (2) all the paths from s to t

example: suppose your code is named bdfs1, then when you execute bdfs1, you should see:

Please input network filename: (then you input, say, test1.sp)
Please input a source node: (then you input, say, 2)
Please input a sink node: (then you input, say, 6)

then you should see the output like

BFS: 2[0] 3[1] 4[1] 5[2] 6[2]
2->6: 4paths
*[2]: 2-4-6
[3]: 2-3-4-6
[4]: 2-3-5-4-6
[3]: 2-3-5-6

The first line printouts the indices of the reachable nodes from s in BFS ordering with [ ] storing its associated distance label. In this example, we know the minimum # arcs from node 2 to node 6 is 2. Note that the BFS ordering is not unique, since it depends on the ordering of the arc adjacency list.
The second line printouts total # paths connecting s and t. Then starting from the third line, you should print out each such path from 2 to 6, with [ ] storing the length of the path. In this example, the shortest path from 2 to 6 is 2-4-6 with 2 arcs (i.e. [2]). Note that there may be more than one shortest path, and you should mark * for each of them.

Likewise, if you input s=3, t=6, it should output 3[0] 5[1] 4[1] 6[2]; 3->6: 3 paths; *[2]: 3-4-6, [3]: 3-5-4-6, *[2]: 3-5-6;
if you input s=4, t=6, it should output 4[0] 6[1]; 4->6: 1 path; *[1]: 4-6;
if you input s=3, t=1, it should output 3[0] 5[1] 4[1] 6[2]; 3->1: 0 path;
if you input s=1, t=3, it should output 1[0] 2[1] 3[1] 4[2] 5[2] 6[3]; 1->3: 2 paths; *[1] 1-2, [2] 1-2-3;

!!Special Notice!!

  1. Please use a data structure similar to the FORWARD STAR (or FORWARD AND REVERSE Star) introduced in the textbook to store your network.

What's difficult in this assignment may be

  1. how you define your data structure (you may require some structures such as a path that stores a total number of arcs in that path and the arc index of that path). Dynamically allocating memory is required to solve this problem (e.g. you do not know how many paths connect s & t in advance).
  2. how you apply the DFS idea to generate all paths. This is the key and MOST DIFFICULT part, and I suggest you try to write down your pseudo-code first, apply your pseudo code manually by drawing pictures, after you are certain what you are doing, then implement your pseudo code.
  3. how you design your data structure: You HAVE TO use something similar to the FORWARD STAR (or FORWARD AND REVERSE Star) In C (or C++), you may have to define a structure for nodes and arcs. You may use an array instead of a pointer to record FirstArc[i] & CurrentArc[i], if you feel using a pointer is troublesome.

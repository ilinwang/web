Programming Requirements
updated at 10/02/23
 

General programming requirements for this course

  • Several programming assignments will be given
  • You should use C or C++ to program (if not, ask me for permission)
  • Whatever editing/debugging software you use to program, you have to make sure your codes can run on Unix environments such as cygwin, Sun (school's UNIX account), Linux, or BSD system

What to turn in for programming assignments:

  • Source code: You should turn in the electronic version

How to do electronic turn in for programming assignments:

  • Your codes have to be put in a directory named rXXXX_YY where rXXXX is your student ID and YY is the name for the assignment (which will be announced later). For example, r3890123_hw1 is the directory for homework #1 by a student with ID r3890123.
  • Then you have to zip the directory in UNIX
    e.g. zip -r r3890123_hw1.zip r3890123_hw1 which will zip the directory r3890123_hw1into a file named r3890123_hw1.zip
  • Then you go to the moodle class web to submit the zipped file before the deadline
    !!Note!! if you are late for the deadline, you will NOT be able to submit your codes, and thus you will get NO grades!! So, I suggest you submit the file at least 30min before the deadline.
  • Any late homework will NOT be graded, except very rare cases for very special reasons.

What you should do in your source code:

  • All of your codes should be in a directory named by the above rules
  • At the beginning of each of your codes, you should write detailed info about that codes, a C/C++ example will be like the following 
    /*
   This code can be compiled and run ok.
or This code can be compiled, but when I run it with input such as ....
     it will give the wrong output such as ....
or This code can NOT be compiled, it has the following errors:....

   This is to convert the algorithm output into excel txt format
   
   usage:
     convt_out1  xxx  yyy
     where xxx is the algorithm output filename, e.g. real_cases.txt
           yyy is the excel txt format filename
  
   input file:
     xxx
	   
   output files:
     yyy_t.txt  : records the alg time
     yyy_n.txt  : records the alg n_tricomp
     yyy_normal_t.txt : normalized alg time
     yyy_normal_n.txt : normalized alg n_tricomp
	   
   compile:
     gcc -o convt_out1 convt_out1.c
   
   pseudocode:
   QUICKSORT(A,p,r)
   ------
   if p<r
     then q=PARTITON(A,p,r);
          QUICKSORT(A,p,q-1);
          QUICKSORT(A,q+1,r);
   ------
   PARTITION(A,p,r)
   ------
   x=A[r];
   i=p-1;
   for j=p to r-1
      if A[j]<=x
         i=i+1;  swap(A[i],A[j]);
   swap(A[i+1],A[r])
   return i+1;
   ------
 
   coded by I-Lin Wang, ID: r38921234, email: wangilin@mail.ncku.edu.tw
   date: 2004.02.23
*/
    In particular, your code has to include detailed English comments [20%] such as
    • whether your code can compile/run or not, if NOT, what's wrong ..etc.[2%]
    • purpose: explain what this code is used for [2%]
    • usage: how to execute this code [2%]
    • input/output file: list any input/output files required [2%]
    • compile: how to compile this code [2%]
    • pseudo-code: briefly illustrate the procedures of your codes [8%]
    • who/when: list who wrote this code and when is it done [2%]
  • Inside the code, you should add English comments so that it is easily readable (understandable) [You may add Chinese comments as you wish, but even so, you should also add English ones.]

General rules about how your programming assignment is graded:

  • 基本分(30%)。只要你的程式足以表現你已費了足夠的精神﹐不論是否能跑﹐就可以得到基本分數。必要的條件是程式必須能通過編譯器（compiler）﹐而且必需具備相當齊全的基本架構。
  • 正確性(25%)。程式是否能正確地執行所應具備的基本功能。
  • 程式架構(15%)。是否適當地將整個程式分成多個函數、函數的分工是否恰當、函數的參數是否恰當、程式是否模組化、函數裡的指令使用是否恰當。有些題目會指定函數的架構。是否依指示設計函數亦列入本項計分。
  • 完整的comment 說明(20%)。請依照這裡的指示來寫你程式的英文comment.
  • 可讀性(10%)。是否有良好的英文註解、變數與函數命名是否具可讀性。
  • 抄襲(-100%)。只要你抄襲程式被抓包，不但該次分數為0分，而且你還會得到該次分數的負分，而且還會被告知系辦公室與學校，可能記過，下場十分悲慘，最好不要以身試法。反之即使你程式寫不正確，但只要按照上面給分方式，通常仍會獲得50%以上的分數，不無小補。

More about codes running in Unix environments:

  • If you use Visual C++ for coding after you are done, you should put everything in a UNIX environment, and compile your codes there, to make sure your codes DO compile/run in a UNIX environment.
  • A convenient UNIX environment for you is your school UNIX account, which is on a Sun machine (or Linux?). See http://cc.ncku.edu.tw/files/11-1255-3313.php for details. If you use this account to compile/run your codes, you have to telnet to the server and do everything on a terminal window, which may not be too convenient.
  • If you want to compile on your own machine, you may
    • install cygwin on Windows, which may occupy 500MB of your HD. It provides a unix-like environment on your Windows. (I have packed a basic cygwin package, see here for details)
    • install Linux or FreeBSD. The computers in our department have installed RedHat Linux which is convenient for you to do programming.
  • Reasons for these unix-compatible requirements:
    • I don't want you to call some built-in functions that Visual C++ or other compiler offers. Instead, I want you to code the functions from the scratch by yourself.
    • I hope each student is at least NOT afraid of the UNIX environment, and at least learns to do things such as telnet, ftp, compiling codes, editing codes, and some basic commands manually. It will help you in the future.
  • If you are not familiar with some UNIX commands, ask your friends/classmates, or come to ask TA or me.

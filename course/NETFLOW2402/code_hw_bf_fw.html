4th coding assignment
Due 05/25 (Mon) 9:00

You are asked to write 2 programs which can

  • read the following network file: test1.sp
  • 1st program (spbf1): use Bellman-Ford implementation (page 141.142)
  • 2nd program (spfw1): use Floyd-Warshall's implementation (page 147.148)
  • both programs allow the user to input a source node index s, then output a shortest path with its length from s to all the other nodes.(i.e. a 1-ALL shortest path problem), together with a number showing how much time it takes to run the main part of the program
  • you should generate at least 1 test problem from each of the following random network generators: (uncompress netflow_splib.rar; in the folder netflow_splib, keyin "make gens", then follow instructions in the folder netflow_splib/doc to learn how to use these generators)
    • spgrid : generate a random grid network
    • sprand : generate a random network
    • spacyc : generate a random acyclic network

Screen Input: network filename (e.g. test1.sp), s (source node)
Screen Output: for each other node, printout a shortest path with its length from s; in the last line, printout the time it takes to run your code

example: suppose your code is named as spbf1(& spfw1), then when you execute spbf1(& spfw1), you should see:

Please input network filename: (then you input, say, test1.sp)
Please input a source node: (then you input, say, 2)

then you should see the output like

2->1: [can not reach]
2->3: [2] 3<-2
2->4: [2] 4<-2
2->5: [4] 5<-3<-2
2->6: [7] 6<-5<-3<-2
time: 2343.22

Likewise, if you input source node as node 3, it should output

3->1: [can not reach]
3->2: [can not reach]
3->4: [1] 4<-3
3->5: [2] 5<-3
3->6: [5] 6<-5<-3
time: 146.16

Note that I do not know what the unit of time is (it is not sec or mili-sec). If you know what its unit is, please lte me know. Anyway, you may just use it as a measure for the practical efficiency of codes.

What's difficult in this assignment may be

  1. how to do Queue operations : (for Bellman-Ford label-correcting algorithm)
    this should be no problem for you since you have done BFS before. Take a closer look at page 142, then you should be able to do it.
  2. how to trace route : (Floyd-Warshall algorithm)
    Floyd-Warshall algorithm is very easy to implement (at most 5-min as I said). What may consume some of your time is how you trace route by the nxn predecessor matrix. This should also not be hard.

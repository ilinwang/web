5th coding assignment
Due 06/22 (Tue) 14:00

This assignment asks you to implement and test six ALL-ALL shortest path codes: DIKH, DIKD, BF, PAPE, Floyd-Warshall (algebraic & graphic).
Each student is assigned to use one network generator for testing about 96 or 100 cases. For each case and each algorithm, you should record the following two parameters :(1) running time obtained by the given function mytimer (2) total number of nontrivial triple comparisons. That is, you should record 6*(# cases) data, where 6 means 6 algorithms (DIKH, DIKD, BF, PAPE, FWA, FWG), and # cases means the total number of randomly generated networks. Each student has to make plots or tables on their computational results.

The report not only has to contain the plots/tables of your results, but also include discussion about why the results are shown as tested. The student sand network generators are assigned as follows:

  1. 黃鈴雅 : spgrid X Y seed -cl10000 -cm1000 -ilc_max -imc_min
    X: 10, 20, 40, 80; Y: 10, 20, 40, 80; seed: 1, 10, 100; (c_min,c_max): (1, 1000), (1000, 10000), (1, 1000000) 共 4x4x3x3=144個 cases
  2. 黃振維 : sprand n m seed -llc_max -lmc_min
    n: 100, 400, 800, 1600, 3200, 6400; m: 4*n, 16*n, n*(n-1)/4; seed: 1, 10, 100; (c_min,c_max): (1, 1000), (1000, 10000), (1, 1000000) 共 6x3x3x3=162個 cases
  3. 黃彥融: spweb2 name x y c1min c1max c2min c2max rmin rmax seed
    x: 10, 100, 1000; y: 10, 50, 100; (c1min,c1max)=(1,1000), (1000,10000); (c2min,c2max)=(1,1000), (1000,10000); (r_min, r_max)=(0.1,1); seed: 1, 10, 100, 1000 共 3x3x2x2x1x4=154個 cases
  4. 葉政霖 : spweb name x y cmin cmax issym seed
    x: 10, 100, 1000; y: 10, 50, 100; (cmin, cmax)=(1,1000), (1000,10000);    issym: 0, 1; seed: 1, 10, 100, 1000 共 3x3x2x2x4=144個 cases
  5. 林瑩婷: spsun name n l cmin cmax issym seed
    n: 50, 100, 200, 400, 800, 1600; l: 1, 4; (cmin, cmax)=(1,1000), (1000,10000);  issym: 0, 1; seed: 1, 10, 100, 1000 共 6x2x2x2x3=144個 cases
  6. 李蕎?: spflower name n1 n2 d2 c1min c1max c2min c2max issym seed
    n1: 50, 100, 200, 400, 800, 1600; n2: n1/2, n1; (c1min,c1max)=(1,1000), (1000,10000); (c2min,c2max)=(1,1000); issym: 0, 1; seed: 1, 10, 100, 1000 共 6x2x2x2x1x3=144個 cases
  7. 曾富隆: spgrid X Y seed -cl10000 -cm1000 -ilc_max -imc_min
    X: 10, 20, 40, 80; Y: 10, 20, 40, 80; seed: 1, 10, 100; (c_min,c_max): (1, 1000), (1000, 10000), (1, 1000000) 共 4x4x3x3=144個 cases

In summary, this is what you should do:

  1. finish 6 ALL-ALL shortest path codes: DIKH, DIKD, BF, PAPE, FWA, FWG. Your codes should be able to let the user define the total number of iteration n_it (i.e. n_loop) so that it can reset the input and rerun the all pairs shortest paths for n_it times in case the running time is too short.
  2. run all cases for all the 6 codes, record the computational time using mytimer() and total number of triple comparisons. (!Note! : if you find the mytimer() gives a very small number due to the small size of the network, you should adjust the n_it so that the running time is measurable.)
  3. you should analyze the results of your computational tests and discuss the tendancy you observe from the results. For example, you may write a little about why which specific algorithm perform better or worse in which case,...etc. Also, you should state the environment of your experiments (e.g. CPU type, RAM size, HD size, OS type, Compiler version...etc)
  4. you should create a folder named as r3890123_HW5 in which you put all the cases ou have tested in a folder named as inputs, put your codes and your reports (written using .doc or whatever text format) in a folder named as reports, compressed the folder of r3890123_HW5 to be r3890123_HW5.zip or r3890123_HW5.rar, and then upload the compressed file to the iteach server. (case檔案因為可能會太大[譬如超過1MB]， 所以不用一併上傳，不過你若有使用任何batch script技巧，必須將batch script一併附上)
  5. you should print out the report and turn in by 06/22 Tue 14:00 [e].

What's difficult in this assignment may be

  1. how to implement FWG
  2. how to do the n_it (i.e. n_loop): note that you should include the reset the main procedures inside the while loop of the n_it
  3. how to write the batch script: you have to do everything in command line. So, you should create some text file, say, kkk.bat, which contains the commands you are going to execute, and then chmod 777 kkk.bat so that whenever you execute kkk.bat it will execute all the commands one by one.
